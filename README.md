# MA-DTSR-Multi-Agent-Delay-Tolerant-Semantic-Routing-
The framework introduces two mechanisms not present in prior semantic routing work. First, a waiting factor embedded in the reinforcement learning action space: at each contact opportunity, an agent may choose not only which neighbour to forward a message to, but whether to wait for a potentially better contact, weighing the TTL cost of waiting against the semantic gain of a more relevant relay. This is a decision structure native to DTN environments, where agents physically carry messages and contact opportunities are opportunistic, yet it has not previously been formalised as an RL action in semantic routing protocols. Second, a diversity-seeking self-organisation mechanism in which agents, over time, maintain neighbourhoods that are deliberately heterogeneous in resource type. Rather than clustering with similar agents, each agent biases its contact table toward nodes holding complementary resources, ensuring that any query type can be served by a direct neighbour or a one-hop relay. This emergent diversity arises from purely local replacement decisions, no global coordination required, and represents a novel application of decentralised facility-location reasoning to dynamic disaster network topology.
The resource representation used throughout MA-DTSR treats each node's available assets as a structured resource descriptor, a fixed-dimensional feature vector encoding resource category, quantity, urgency, mobility type, location zone, energy level, and agency affiliation. Similarity between a query and a descriptor is computed using configurable distance metrics including L1 (Manhattan) and Euclidean distance, enabling comparison across heterogeneous resource vocabularies without requiring a shared ontology. The representation is explicit, interpretable, and designed to be transmitted in constrained message formats compatible with structured data encodings such as XML or binary serialisation. The choice of representation is itself a variable in the framework: different formats induce different similarity geometries, and the protocol is designed to accommodate this variation.
The specific contributions of this work are as follows.
1.	A formal system model (§3) linking per-hop routing decisions to success probability, message cost, and mission utility through explicit equations, providing a mathematical foundation for all protocol decisions.
2.	A reinforcement learning extension (§3.8) in which the next-hop decision is governed by a learned Q-function shaped by mission utility, with a two-action space, forward or wait, that captures the DTN-specific contact timing problem.
3.	A cooperative self-organisation mechanism (§3.6) in which agents exchange learned Q-values with adjacent contacts and maintain diversity-seeking neighbourhood tables, enabling emergent global coordination from purely local interactions.
4.	A configurable semantic resource representation (§3.9) comparing structured feature vectors under multiple similarity metrics, with an analysis of how representation choice affects routing behaviour.
5.	A simulation study (§5) evaluating MA-DTSR against epidemic routing [3], uninformed random walk, and a heuristic-only semantic baseline across network sizes N ∈ [100, 300], TTL budgets TTL ∈ {10, 20, 30, 40}, and varying resource densities, measuring success rate, hop count, message volume, semantic match error, and mission utility.
